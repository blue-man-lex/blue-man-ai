# blue-man-ai
Интеграция Gemini ИИ для NPC: Живые диалоги, Торговля, Квесты и Оживление мира.

📂 ПАСПОРТ ФАЙЛОВОЙ СТРУКТУРЫ: BLUE MAN AI

### 1. Ядро (Core) — `scripts/core/`
Это мозг и нервная система модуля.
*   **`main.js`**: Входная точка. Инициализация хуков (`init`, `ready`, `renderChatMessage`), регистрация настроек, патчи интерфейса (Token HUD). Здесь мы ловим сообщения-команды.
*   **`settings.js`**: Регистрация всех настроек модуля (API ключи, модели, галочки механик).
*   **`ai-logic.js`**: "Мозг". Формирует промпт для Gemini, очищает текст журналов, учитывает репутацию и отправляет запрос на Google API.
*   **`npc-collector.js`**: "Сборщик". Сканирует актера (био, статы, инвентарь, деньги, боевые параметры) и упаковывает в JSON для ИИ.
*   **`flag-handler.js`**: "Коммутатор". Самый важный файл для синхронизации. Позволяет игрокам (у которых нет прав) менять данные NPC через скрытые сообщения чата, которые перехватывает ГМ.

### 2. Интерфейс и Логика Диалогов (Systems/UI) — `scripts/systems/ui/`
Всё, что видит игрок при общении.
*   **`dialog-window.js`**: Логика главного окна (`BlueManDialog`). Рендер шаблона, обработка ввода текста, управление историей чата.
*   **`button-handler.js`**: Обработчик кнопок внутри окна. Атаки, социальные броски, передача предметов, кнопки ГМа (выдача квеста/награды).
*   **`magic-handler.js`**: Логика кнопки "Магия". Сканирует спеллбук игрока на наличие социальных заклинаний (Очарование и т.д.) и запускает их.
*   **`steal-dialog.js`**: Мини-игра карманной кражи. Окно со списком лута NPC и расчетом сложности (DC).
*   **`markers.js`**: Отрисовка иконок `!` и `?` над головами токенов через PIXI (визуальный слой).

### 3. Система Квестов (Systems/Quest System) — `scripts/systems/quest-system/`
Полноценный квест-лог внутри модуля.
*   **`quest-config.js`**: Окно ГМа для привязки журнала к NPC (Drag-and-Drop зона).
*   **`quest-log.js`**: Окно "Журнал Заданий" для игроков. Группировка, статусы, сворачивание.
*   **`quest-service.js`**: Работа с базой данных квестов (World Settings). Парсинг наград (деньги/предметы) из текста журнала.
*   **`reward-dialog.js`**: Окно "Сундук". Визуализация награды, которую можно перетащить или кликнуть, чтобы забрать.

### 4. Оживление Мира (Systems/Ambient System) — `scripts/systems/ambient-system/`
*   **`ambient-logic.js`**: Цикл проверки дистанции. Если игрок близко — NPC кидает фразу над головой (пузырь).
*   **`ambient-phrases.js`**: База заготовленных фраз на случай, если ИИ отключен или лимиты исчерпаны.

### 5. Интеграции (Systems/Integrations) — `scripts/systems/integrations/`
*   **`patrol.js`**: Связка с модулем *Patrol*. Если патрульный замечает игрока — ИИ генерирует окрик.
*   **`compatibility.js`**: Фиксы конфликтов с другими модулями (например, перехват `console.warning`).

### 6. Шаблоны (HTML) — `templates/`
*   `dialog.html`: Основное окно чата (3 колонки: Игрок, Чат, NPC).
*   `quest-config.html`: Настройки NPC для ГМа.
*   `quest-log.html`: Красивый журнал заданий.
*   `reward-dialog.html`: Окно выдачи награды.

### 7. Стили — `styles/style.css`
*   Все визуальные стили: шрифты (*Modesto, Signika*), анимации, верстка окон.
