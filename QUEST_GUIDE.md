# Руководство по архитектуре квестов: Blue Man AI

Эта инструкция описывает ПРАВИЛЬНУЮ иерархию папок в Журнале Foundry VTT и систему грейдов сложности.

---

## 📂 Глобальная структура (Пример: Глава 1)

Создайте основную папку для главы или акта игры. Внутри неё иерархия должна выглядеть так:

### 1. 🔵 ПАПКА "ДОСКА" (Notice Board)
**Расположение:** `[Корневая папка] / Доска`
**Тип квестов:** Контракты и заказы, которые игроки "срывают" с доски.
**Цвет в UI:** Голубой / Синий.

*   **Как работает:** Все журналы, лежащие прямо в этой папке, отображаются на Доске объявлений.
*   **Важно:** Для этих квестов ОБЯЗАТЕЛЬНО используйте префиксы сложности (см. ниже).

---

### 2. 📁 ПАПКИ РЕГИОНОВ (Например: Ривермун)
**Расположение:** `[Корневая папка] / [Имя Региона]`

Внутри папки региона работает автоматическое разделение на Главные и Побочные квесты:

#### 🔴 ГЛАВНЫЕ КВЕСТЫ РЕГИОНА (Main Region Quests)
*   **Где лежат:** Прямо в папке региона (например, в `Ривермун`).
*   **Цвет в UI:** Красный.
*   **Суть:** Это основные задачи, определяющие прогресс в этой локации.

#### 🟡 ПОБОЧНЫЕ КВЕСТЫ / КВЕСТЫ НПС (Side Quests)
*   **Где лежат:** В специальной подпапке `квесты НПС` внутри папки региона.
*   **Цвет в UI:** Желтый.
*   **Суть:** Личные просьбы жителей или мелкие поручения.

---

## 🏆 СИСТЕМА ГРЕЙДОВ (Сложность)

Модуль автоматически определяет сложность квеста по первой букве в названии Журнала. Это влияет на визуальный стиль и заголовок на Доске Объявлений.

| Префикс | Грейд | Сложность | Стиль в UI |
| :--- | :--- | :--- | :--- |
| **[S]** или **[С]** | Rank S | **Смертельный** | Самый высокий приоритет, яркий акцент |
| **[A]** или **[А]** | Rank A | **Сложный** | Высокая сложность |
| **[B]** или **[Б]** | Rank B | **Средний** | Стандартная сложность |
| **[C]** или **[Ц]** | Rank C | **Легкий** | Простая задача |
| **[D]** или **[Д]** | Rank D | **Обычный** | По умолчанию (если префикс не указан) |

## 📝 Пример оформления квеста (с наградами)

Чтобы система предложила ГМу выдать награды, используйте формат `@UUID` для предметов и `gp/sp/eb` для валюты.

```html
<p># [B] Охота на Гноллов</p>
<p>**Источник:** Капитан Стражи (Форт "Пыльный")</p>
<p>**Статус:** Опасно</p>

<h3>Описание</h3>
<p>Караваны, идущие через Пустоши, пропадают один за другим...</p>

<h3>Задачи</h3>
<p>1. Найти лагерь гноллов в каньоне.</p>
<p>2. Уничтожить вожака стаи.</p>

<h3>Награда</h3>
<p>**Награда:** 150gp, 50sp, @UUID[Compendium.dnd5e.items.Item.5m9ErO9In8Uc5yyf]{Гигантское зелье лечения}</p>
```
для Доски 
[C] Сбор Луноцвета
<p>Алхимик Силантрия срочно ищет помощников с острым зрением.</p>
<p>Для создания важного зелья мне требуется ровно пять цветков Луноцвета. Они растут только на восточной опушке леса и распускаются исключительно в полночь.</p>
<p><em>Осторожно: ночью в лесу могут бродить дикие волки.</em></p>
<p><strong>Награда:</strong> 25sp и три малых зелья лечения.</p>

## 🛠️ Как привязать NPC к этой иерархии?

1. Откройте диалог с NPC.
2. Нажмите **⚙️ (Настройки)**.
3. Перетащите нужный квест из Журнала в поле **"Привязанный квест"**.
4. Теперь ИИ будет знать все детали этого задания, его грейд и награду, и будет использовать это в разговоре.
